# Manan Mehta Portfolio (Astro / Codefolio)

Recruiter-focused static portfolio built on the `danielunited/codefolio` Astro template.

## Stack
- Astro 5 (static output)
- Centralised content in `src/data/portfolio.ts`
- Template styling in `public/assets/styles.css`

## Setup
```bash
npm install
npm run dev
```

## Build and Preview
```bash
npm run build
npm run preview
```

## Deploy
This project is static and can be deployed to Vercel, Netlify, or GitHub Pages.

## Content Update Guide
Edit only these files for content changes:

- `src/data/portfolio.ts`
- `public/assets/MananMehta_Resume.pdf`

Key sections in `src/data/portfolio.ts`:
- `site`: SEO title, description, canonical URL, OG image
- `hero`: hero copy and CTA links
- `about`: about bullets
- `skills`: grouped skills with proficiency
- `experience`: work history bullets
- `projects`: featured projects (problem, approach, stack, impact, links, highlights)
- `education`: degree history and modules
- `certifications`, `languages`, `contact`

## SEO and Crawling
- Canonical metadata is set in `src/components/Header.astro`
- `robots.txt` is generated by `src/pages/robots.txt.ts`
- `sitemap.xml` is generated by `src/pages/sitemap.xml.ts`

## Deployment Checklist
1. Replace `https://TODO-your-domain.com` in `src/data/portfolio.ts` and `astro.config.mjs`.
2. Replace all `TODO:` project/social links in `src/data/portfolio.ts`.
3. Optionally replace `public/assets/profile-placeholder.svg` with your real headshot.
4. Run `npm run build` and confirm no errors.
5. Deploy:
   - Vercel: import repo and use default Astro static settings.
   - Netlify: build command `npm run build`, publish directory `dist`.
   - GitHub Pages: build to `dist` and publish via Actions.
